## Теоретическая часть

### Обратная польская нотация (ОПН)
Обратная польская нотация (Reverse Polish Notation) - это математическая нотация, в которой операторы следуют за своими операндами. Например:
- Инфиксная запись: `2 + 3`
- ОПН: `2 3 +`

### Алгоритм сортировочной станции (Shunting Yard)
Алгоритм, разработанный Эдсгером Дейкстрой, для преобразования инфиксной записи в ОПН. Основные правила:
1. Операнды сразу добавляются в выходную строку
2. Операторы помещаются в стек с учетом приоритета
3. Открывающие скобки помещаются в стек
4. При встрече закрывающей скобки выталкиваются операторы до открывающей скобки

* **Цель:** Написать калькулятор на Python, прозводящи вычисления с помощью перевода входного выражения в RPN(обратную(постфиксную) польскую нотацию), используя для этого алгоритм шунтирующего двора(Shunting Yard)
* **Библиотеки:** *os*, *re*, *sys*, *pytest*
* **Допущения:**
  - Выражение может содержать любое количество пробелов между числами и операторами, поскольку они игнорируются.
  - Любые символы кроме "0123456789-+*/()%. " в выражении запрещены
  - Унарный '-' заменяется на '~'
  - Унарный '+' игнорируется, из исходного выражения убирается
  - Если перед унарной операцией следовала другая операция или унарный оператор заключен в скобки с операндом, к которому он применен, разницы не будет, но лучше записывать в скобках для читаемости выражения
  - Число в программе всегда представляется в виде числа типа float, округляется только результат
  - Операции '//' и  '%' запрещены для вещественных чисел
  - 0**0 не определен
